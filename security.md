# Security Assessment and Policies

## Intended Users
This repository just contains academic coursework. The intended users of the code and data within this repository are really just myself and the professor.

## Assessment of Security Risks
Because this is a repo with academic work, the primary threat is a breach of academic integrity. If the codebase were to fall into the "wrong hands" (e.g. other students), they could plagiarize my asinine code which is a big no-no. 

## Repository Security Measures
To mitigate the risks outlined above, I've done absolutely nothing. Nada.

Don't get me wrong, I do think data integrity and security are vital, I just truly have nothing worth protecting in this particular repo.

If someone does get "unauthorized access" (which would mean they'd be reading this file as well), then my message to you the tresspasser would simply be: why?
